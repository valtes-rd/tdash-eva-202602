# T-DASHエバンジェリスト認定試験 練習用サロン予約アプリケーション

第4回T-DASH認定試験向けのテスト自動化練習用シングルページアプリケーション（SPA）です。Vite + React（HashRouter）で構築しており、GitHub Pagesへデプロイできる構成になっています。バックエンドは持たず、全ての処理をクライアントサイドで完結させます。

## 🌐 公開URL
https://valtes-rd.github.io/tdash-eva-202602/

## 主な機能
- ログイン（登録済みユーザーのみ / 認証失敗時はエラー表示）
- 共通ヘッダー（サロン一覧、マイページ、予約一覧、お気に入り、ログアウト）
- サロン一覧（カテゴリ絞り込み、日付指定で空き状況更新、詳細は別タブ、空きがある場合のみ予約ボタン有効）
- サロン詳細（別タブ表示、直近1週間の空き状況、お気に入り登録/解除）
- 予約登録（重複予約防止、モーダル確認→確定→予約一覧へ遷移）
- 予約一覧（利用済み表示・翌日以降のみキャンセル可、確認モーダル経由）
- マイページ（ログインユーザー情報表示）
- お気に入り一覧（登録サロンのみ表示、解除操作はLoading表示付き）

## 🧪 テストシナリオのポイント

### 1. ログイン機能
- 複数のテストユーザーでログインが可能
- パスワード間違いでエラーメッセージ表示
- ログイン成功後、マイページでユーザー名確認可能

### 2. ログアウト機能
- JavaScript confirmダイアログ使用
- OKで実行、キャンセルで中止

### 3. サロン詳細
- 別ウィンドウ（target="_blank"）で開く
- お気に入り登録/解除が可能（非同期処理）

### 4. 予約機能
- 日付選択により空き状況が動的に変化
- 特殊ルール（定休日、満員日）が適用
- 予約確認モーダル表示
- 重複予約防止機能

### 5. 非同期処理
- Loading表示（loading...）
- 500-2000msのランダム遅延
- 画面操作ブロック

## 実装詳細
- 初期データは `src/data/mockData.js` に定義し、React Stateに読み込みます。ブラウザのメモリのみで管理し、リロードすると初期状態に戻ります。
- 疑似通信のために `src/data/fakeFetch.js` を利用し、500〜2000msのランダム遅延とローディングオーバーレイを表示します。
- ルーティングは GitHub Pages 対応の `HashRouter` を使用しています。
- デプロイ用に `package.json` の `homepage` と `deploy` スクリプト（gh-pages）が設定されています。
